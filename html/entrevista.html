ENTREVISTAS

La forma mas obvia de obtener información acerca del sistema es directamente hablando con el cliente, en esta el entrevistador se encarga de realizar preguntas acerca del sistema y las documenta, esto permite tener una mejor relación con el cliente. El cliente describe una necesidad y el desarrollador interpreta los requerimientos y analiza su factibilidad, el cliente describe éstos requerimientos de forma general y no de forma funcional. El desarrollador debe de contemplar estos requerimientos dados por el cliente para clasificarlos en funciones y restricciones.
